# Contributing

Thank you for your interest in contributing to Kings Theorem.

Guidelines
- Open an issue to discuss significant changes before implementing.
- Fork the repository and create feature branches from `develop`.
- Create pull requests targeting `develop` and request review from maintainers.
- Write tests for new features and run the test suite locally before opening a PR.
- Keep secrets, keys, logs, and large model artifacts out of the repo.

Code style
- Follow existing project style. Use linters and formatters where applicable.

Security
- Do not commit credentials, private keys, or `.env` files. These are ignored by `.gitignore`.
